H-847. 메모리 = DATA 이산 저장의 CAS 읽기/쓰기

핵심 발견

H-847: 메모리 = DATA 이산 저장의 CAS 읽기/쓰기

$$\text{DATA}[addr] \xrightarrow{\text{CAS read}} \text{output} \;;\quad \text{input} \xrightarrow{\text{CAS write}} \text{DATA}[addr]$$

등급: B

메모리는 DATA를 이산 주소에 저장하고 CAS를 통해 읽기/쓰기하는 장치이다. 각 메모리 셀 = DATA 단위 저장소. 주소 = ECS 좌표. 읽기 = CAS Compare 후 DATA 복사. 쓰기 = CAS Swap으로 DATA 덮어쓰기.

공리 4(DATA 이산), 공리 5(CAS 읽기/쓰기), 공리 2(ECS 주소 격자)

공리 4: DATA는 이산 단위로 저장된다. 1비트 = 최소 DATA 단위. 바이트/워드 = DATA 묶음. 공리 5: CAS Compare = 읽기(DATA 비파괴 복사), CAS Swap = 쓰기(DATA 갱신). 공리 2: ECS 격자가 주소 공간을 제공. 물리 주소 = ECS 좌표.

DRAM: 커패시터 전하 = DATA 저장. 주기적 리프레시 필요(CAS 반복). SRAM: 래치 = 안정 DATA 저장. 리프레시 불필요. Flash: 부유 게이트 전하 = 비휘발 DATA. HDD/SSD: 자기/전하 = 대용량 DATA.

DRAM: $\sim 16$~$64\;\text{GB}$. 접근 시간: $\sim 10\;\text{ns}$. SSD: $\sim\;\text{TB}$. 접근 시간: $\sim\;\mu\text{s}$. 레지스터: $\sim\;\text{ps}$ 접근.

메모리 계층 구조 지연시간 비율 이론과 실측 정합.

DRAM, SRAM, Flash, 메모리 계층, 폰 노이만 아키텍처

컴퓨터 메모리 기술 70년 이상의 실증.

CAS 읽기/쓰기 최소 에너지의 란다우어 한계 연결.

재대입 용도: 재대입 용도: H-845(집적회로). H-844(트랜지스터). H-862(광통신)
